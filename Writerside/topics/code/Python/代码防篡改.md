# 代码防篡改

> 突然想起之前看过的知乎故事，大概就是程序员遭到诬陷，要公开审阅他的代码，但这名程序员给自己的代码加设过防篡改，幸免遇难。
>

突然间也想试试，虽然不至于是一坨屎，但也没有多少价值。但就是按耐不住内心的好奇，所以问 AI 要了一个案例

## 说明

基于 RSA 非对称加密，实现代码添加专属加密签名、防篡改、身份标识全套流程，特点如下：

- 原代码无需修改、可直接正常运行；
- 密钥仅生成一次，所有代码文件通用；
- 篡改代码任意字符，验证都会失败；
- 无私钥无法伪造签名，可快速识别代码归属。

## 核心概念

- 私钥（private_key.pem）：个人专属，严禁对外泄露，用于给代码生成签名。
- 公钥（public_key.pem）：可公开分发，用于验证签名、校验代码完整性。
- 签名文件（signature.txt）：对应单个代码文件的加密标识，用于身份核验与防篡改。
- 规则：一套密钥 = 个人专属身份，所有代码文件共用，无需重复生成密钥。

### 生成密钥对

```python
from cryptography.hazmat.primitives.asymmetric import rsa
from cryptography.hazmat.primitives import serialization

# 生成 RSA 密钥对
private_key = rsa.generate_private_key(public_exponent=65537, key_size=2048)

# 保存私钥（自行妥善保管）
with open("private_key.pem", "wb") as f:
    f.write(private_key.private_bytes(
        encoding=serialization.Encoding.PEM,
        format=serialization.PrivateFormat.PKCS8,
        encryption_algorithm=serialization.NoEncryption()
    ))

# 保存公钥（可公开）
public_key = private_key.public_key()
with open("public_key.pem", "wb") as f:
    f.write(public_key.public_bytes(
        encoding=serialization.Encoding.PEM,
        format=serialization.PublicFormat.SubjectPublicKeyInfo
    ))

print("✅ 密钥生成完成！")
print("🔒 private_key.pem（私钥，禁止外传）")
print("🔓 public_key.pem（公钥，可公开）")
```

运行后，生成公钥 (public_key.pem) 和私钥 (private_key.pem)

### 生成文件签名

> 每个文件都需要单独签名
> 
> 私钥不需要更换，但如果需要更换那么之前的公钥、签名全部作废

要求生成签名时需要私钥、签名的文件放置在同一目录

```Python
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.asymmetric import padding
from cryptography.hazmat.primitives import serialization
import base64

# ========== 请手动修改此处配置 ==========
CODE_FILE = "main.py"        # 你的代码文件名
PRIVATE_KEY = "private_key.pem"
SIGNATURE_FILE = "signature.txt"
# ======================================

# 加载私钥
with open(PRIVATE_KEY, "rb") as f:
    key = serialization.load_pem_private_key(f.read(), password=None)

# 读取代码原始内容
with open(CODE_FILE, "rb") as f:
    code = f.read()

# 生成加密签名
sig = key.sign(
    code,
    padding.PSS(mgf=padding.MGF1(hashes.SHA256()), salt_length=padding.PSS.MAX_LENGTH),
    hashes.SHA256()
)

# 写入签名文件
with open(SIGNATURE_FILE, "w", encoding="utf-8") as f:
    f.write("# 代码专属加密签名\n# 归属：个人原创代码\n")
    f.write(base64.b64encode(sig).decode())

print(f"✅ 签名生成完成！签名文件：{SIGNATURE_FILE}")
```

### 验证文件

代码文件 + public_key.pem + 对应 signature.txt 放置在同一目录

```python
from cryptography.hazmat.primitives.asymmetric import padding
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives import serialization
import base64

# ========== 请手动修改此处配置 ==========
CODE = "main.py"        # 待验证的代码文件
PUBLIC_KEY = "public_key.pem"
SIGNATURE = "signature.txt"
# ======================================

# 加载公钥
with open(PUBLIC_KEY, "rb") as f:
    key = serialization.load_pem_public_key(f.read())

# 读取代码与签名
with open(CODE, "rb") as f:
    code = f.read()

with open(SIGNATURE, "r", encoding="utf-8") as f:
    sig_b64 = f.readlines()[-1].strip()
sig = base64.b64decode(sig_b64)

# 执行验证
try:
    key.verify(sig, code,
        padding.PSS(mgf=padding.MGF1(hashes.SHA256()), salt_length=padding.PSS.MAX_LENGTH),
        hashes.SHA256()
    )
    print("✅ 验证通过：原版代码，未被篡改")
except:
    print("❌ 验证失败：代码被篡改 或 非原创代码")
```
