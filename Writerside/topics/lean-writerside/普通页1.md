# 普通页 1

## seealso

seealso 的作用很简单，就是增加链接，添加与当前主题相关的主题和外部资源的链接。

而让其不报错反而会麻烦，毕竟涉及到 <path>c.list</path>

<tabs>
<tab title="topic文件">

```XML

<seealso>
    <category ref="wrs">
        <a href="https://www.jetbrains.com/help/writerside/markup-reference.html">Markup reference</a>
    </category>
    <category ref="md">
        <a href="https://markdown.com.cn/">markdown</a>
        <a href="https://spec.commonmark.org/0.30/">commonmark</a>
    </category>
</seealso>
```

<deflist>
<def title="title">使用 title 属性可以将 see also 修改为其他文本</def>
</deflist>

</tab>
<tab title="c.list文件">

```xml

<categories>
    <category id="wrs" name="Writerside documentation" order="1"/>
    <category id="md" name="Markdown documentation" order="2"/>
</categories>
```

</tab>
</tabs>

![seealso](seealso.png)

seealso 的用途是指向在当前文档外的其他相关文档、链接，不能指向使用 seealso 的文档

得益于 seealso 的渲染位置，其他标签放置在 seealso 之后不会影响渲染，seealso 依然会在文档的尾部渲染

## chapter

如果不设置该部分，右侧标题栏只有文档提供的 H1 主标题，该标签用于提供效果等同于 H2 的标题

> Writerside 渲染的文档目录只支持 H1 标题和 H2 标题

```XML

<chapter title="标题名称" id="chapter1">
    <p>Whenever you are not sure what to do</p>
    <p>try a barrel roll first</p>
    <p>then ask for help.</p>
</chapter>
<chapter title="标题名称2" id="chapter2"/>
<chapter title="Do a barrel roll" id="barrel-roll">
<title>标题名称3</title>
</chapter>
```

`<chapter>` 通过子元素 `<title>` 修改标题名称。感觉没什么用，不如直接在`<chapter>`中设置好

## 选项卡

### tabs

`<tabs>` 可以将相关的内容组分成可切换的选项卡，以针对不同操作系统、语言、角色等的指令。

一旦创建 `<tabs>` 必须包含至少两个`<tab>`，每个 `<tab>` 都有一个 title 属性：

```XML

<tabs>
    <tab title="tab1">选项卡内容1</tab>
    <tab title="tab2">选项卡内容2</tab>
</tabs>
```

<tabs>
    <tab title="tab1">选项卡内容1</tab>
    <tab title="tab2">选项卡内容2</tab>
</tabs>

还可以使用 `<code-block>` 来代替 `<tab>`，此时 `<code-block>` 的 lang 属性会替代`<tab>`的 title 属性显示

```XML

<tabs>
    <code-block lang="java">
        public static void main(String[] args) {
        System.out.println("Hi!");
        }
    </code-block>
    <code-block lang="python">
        if __name__=="__main__":
        print("Hi")
    </code-block>
</tabs>
```

<tabs>
    <code-block lang="java">
    public static void main(String[] args) {
        System.out.println("Hi!");
    }
    </code-block>
    <code-block lang="python">
    if __name__=="__main__":
        print("Hi")
    </code-block>
</tabs>

### switcher-key

`<switcher-key>` 可以理解为范围更大的`<tabs>`，它会在页面顶端生成一个选择列表，选择不同的选项整个页面都会切换到对应的内容

Writerside 对其进行了 markdown 的实现，使用 markdown 也能实现这一功能，看起来更加简洁方便。
<tabs>
<tab title="Topic">

```xml

<topic title="Topic title" switcher-label="Language">
    ...
    <chapter title="Examples" switcher-key="Java">
        <p>Some Java examples.</p>
    </chapter>
    <chapter title="Examples" switcher-key="Kotlin">
        <p>Some Kotlin examples.</p>
    </chapter>
    ...
</topic>
```

在 chapter 标记内的内容都将随 key 的修改而变化
</tab>
<tab title="Markdown">

```markdown
---
switcher-label: Language
---

# Topic title

...

## Section One {switcher-key="Java"}

Some Java examples.

## Section Two {switcher-key="Kotlin"}

Some Kotlin examples.
...
```

> 在顶部设置 label，在 H2 标题设置 key

</tab>
</tabs>

## deflist & procedure

`<deflist>`是带有相应定义 (描述) 的术语列表

`<procedure>`是带边框和标题的内容框，虽然官方定义是带步骤的列表

<tabs>
<tab title="deflist">

```XML

<deflist>
    <def title="术语">术语描述</def>
</deflist>
```

<deflist> 
    <def title="术语">术语描述</def>
</deflist>
